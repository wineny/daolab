## 1. Overview

### 1.1 Design Goals

1. **스레드 대화**: 최초 @멘션 후 스레드 내에서 멘션 없이 자연스럽게 대화 이어가기
2. **멤버 매칭**: "AI 관심 있는 사람?" 같은 자연어 질문에 관련 멤버 추천
3. **지식 수집**: 멤버가 공유한 자료를 봇 지식으로 축적
4. **지식 관리**: 수집된 지식의 구조적 저장 및 조회

### 1.2 Design Principles

- **OpenClaw 네이티브 활용**: 추가 인프라 없이 OpenClaw 기능만으로 구현
- **최소 설정 변경**: `openclaw.json` 수정 최소화, SOUL.md/지식파일 중심
- **비파괴적 변경**: 기존 v1 기능에 영향 없이 추가

## 3. Feature 1: 스레드 대화 지원

### 3.1 구현 방식

OpenClaw Discord `threadBindings` 설정 활성화로 구현.

### 3.2 설정 변경 (`openclaw.json`)

`channels.discord`에 다음 추가:

```json
{
  "channels": {
    "discord": {
      "threadBindings": {
        "enabled": true,
        "idleHours": 24,
        "maxAgeHours": 0
      }
    }
  }
}
```

| 설정 | 값 | 설명 |
|------|---|------|
| `enabled` | `true` | 스레드 바인딩 활성화 |
| `idleHours` | `24` | 24시간 비활성 시 자동 해제 |
| `maxAgeHours` | `0` | 최대 수명 무제한 (수동 해제까지 유지) |

### 3.3 사용자 흐름

```
1. 사용자: 채널에서 "@Daolab 다오랩이 뭐야?"
2. 봇: 답변 (스레드 자동 바인딩)
3. 사용자: 스레드에서 "그럼 7기는 뭐해?" (멘션 없이)
4. 봇: 답변 (같은 세션 유지)
5. 24시간 비활성 → 자동 해제
```

### 3.4 기존 동작과의 차이

| 항목 | v1 (현재) | v2 (변경 후) |
|------|----------|------------|
| 채널 메시지 | @멘션 필요 | @멘션 필요 (동일) |
| 스레드 메시지 | @멘션 필요 | 멘션 없이 대화 가능 |
| DM | 자유 대화 | 자유 대화 (동일) |
| 세션 유지 | 없음 | 스레드 단위 세션 유지 |

### 3.5 필요 작업

- `openclaw.json` 수정 → `openclaw gateway restart` 필요

## 5. Feature 3: 사용자 제공 지식 수집

### 5.1 구현 방식

SOUL.md에 **지식 수집 규칙**을 추가하고, `memory/` 폴더에 수집된 지식을 카테고리별 파일로 저장.

### 5.2 수집 트리거 패턴

| 트리거 | 동작 | 예시 |
|--------|------|------|
| `@Daolab 기억해줘 [텍스트]` | 텍스트 내용 저장 | "기억해줘 다음 모임은 강남역에서야" |
| `@Daolab 이거 기억해 [링크]` | 링크 + 설명 저장 | "이거 기억해 https://..." |
| `@Daolab 저장해줘 [내용]` | 동의어 트리거 | "저장해줘 발표 자료..." |
| `@Daolab 뭐 기억하고 있어?` | 수집 목록 조회 | 카테고리별 수집 현황 표시 |
| `@Daolab [키워드] 기억나?` | 특정 지식 검색 | "강남 모임 기억나?" |

### 5.3 저장 구조 (`memory/` 폴더)

```
workspace-daolab/memory/
├── shared_links.md         # 수집된 링크
├── shared_knowledge.md     # 수집된 텍스트/정보
└── shared_files.md         # 수집된 파일 정보
```

#### `shared_links.md` 형식

```markdown
# 수집된 링크

## 항목
- [2026-03-17] @닉네임: 설명 - https://example.com
- [2026-03-18] @닉네임: 다오랩 블로그 - https://daolab.io/blog
```

#### `shared_knowledge.md` 형식

```markdown
# 수집된 지식

## 항목
- [2026-03-17] @닉네임: 다음 모임은 강남역 스타벅스에서 진행
- [2026-03-18] @닉네임: 7기 MT는 4월 셋째주 예정
```

#### `shared_files.md` 형식

```markdown
# 수집된 파일 정보

## 항목
- [2026-03-17] @닉네임: 발표자료.pdf - 3기 거버넌스 연구 결과 발표
```

### 5.4 SOUL.md 추가 규칙

```markdown
## 지식 수집 규칙
1. "기억해줘", "이거 기억해", "저장해줘" 패턴이 감지되면:
   - 내용을 파악하고 적절한 memory/ 파일에 추가
   - 링크가 포함되면 → memory/shared_links.md
   - 텍스트 정보면 → memory/shared_knowledge.md
   - 파일 첨부면 → memory/shared_files.md
   - 저장 형식: `[날짜] @보낸사람: 내용`
   - 확인 응답: "기억했어! ✅ [요약]"

2. "뭐 기억하고 있어?", "기억 목록" 요청 시:
   - memory/ 폴더의 파일들을 읽어 카테고리별 요약
   - 최근 5개 항목 위주로 표시
   - "더 보고 싶으면 '링크 기억 보여줘' 같이 카테고리를 말해줘!"

3. "[키워드] 기억나?" 요청 시:
   - memory/ 파일에서 키워드 검색
   - 관련 항목 표시

4. 수집 보안:
   - 수집된 내용은 다오랩 관련 질문에만 활용
   - 개인 민감 정보(전화번호, 주소 등)는 수집 거절
   - "그건 민감한 정보라 저장하지 않는 게 좋겠어!"
```
