## Part 1: 블록체인의 핵심 특징

### 1. Immutable (불변성)
- 블록체인에 한 번 기록되면 절대로 수정되지도, 지워지지도 않음
- 데이터가 덮어쓰기 되지 않고 계속 뒤로 추가됨 (로그 방식)
- 이 특성이 블록체인에 **신뢰(Trust)**를 부여하는 핵심 역할

### 2. Open (개방성/투명성)
- 누구나 허가 없이 투명하게 모든 데이터를 볼 수 있음
- 특정 주소를 검색하면 해당 주소의 과거 모든 행적이 공개됨
- 프라이버시 관련: 주소 자체는 익명(Anonymous)이지만, 주소와 실제 신원이 매핑되면 프라이버시가 완벽하지 않음
- 프라이버시 보강 기술이 연구되고 있으며, 이를 적용한 블록체인들이 등장 중

### 3. Shared (공유성)
- 일종의 데이터베이스/데이터 저장소 (구글 드라이브와 유사한 개념)
- 누구나 허가 없이 쓰거나 읽을 수 있음
- 로그인, 허가, 돈 지불 없이도 접근 가능

### 4. Permissionless (무허가성)
- 이미 만들어진 서비스를 다른 서비스가 호출하는 데 아무 제약이 없음
- API 키, 토큰당 비용 등이 불필요
- 서비스 조합(Composability)이 매우 쉬움

### 5. 분산 디지털 장부 (Distributed Ledger)
- **Ledger(원장)**: 은행 등에서 회계 기록을 하는 것
- 아무 데이터나 들어가는 것이 아닌, 중요한 **트랜잭션(Transaction)** 데이터가 기록됨
- 블록체인은 영화나 파일을 저장하는 용도가 아님
- 기본적으로 트랜잭션, 아이덴티티 정보 등 중요한 텍스트 정보가 저장됨

### 블록체인의 혜택 (Benefits)
- **Trust (신뢰)**: 한 번 기록되면 수정 불가 + 누구나 열람 가능 = 신뢰 엔진
- **Transparency (투명성)**: 모든 기록이 공개
- **Trust Engine (신뢰 엔진)**: 서로 못 믿는 엔티티들 간의 중간 시스템으로 적합
- 계약서를 블록체인에 올리면, 종이 계약서나 디지털 파일보다 훨씬 신뢰할 수 있음

## Part 2: 블록체인 기술 구조

### 탈중앙화 노드 구조
- 네이버나 카카오처럼 하나의 회사가 운영하는 것이 아님
- 전 세계 수만 대의 노드가 분산되어 운영
- 누구나 프로그램을 다운로드 받아 노드를 돌릴 수 있음
- 비트코인/이더리움 중단 방법: **모든 노드를 동시에 종료**해야 함
  - 6만 대 중 1대만 살아있어도 네트워크는 계속 작동
- **검열 저항성(Censorship Resistance)**: 어떤 나라도, 어떤 회사도 블록체인을 통제할 수 없음

### 트랜잭션 처리 과정
1. **앨리스**가 **밥**에게 1 비트코인을 보내겠다는 트랜잭션 생성
2. 앨리스의 **프라이빗 키**(금고 열쇠)로 서명
3. 트랜잭션을 블록체인 네트워크에 전송
4. 블록체인에서 검증:
   - 앨리스가 보낸 것이 맞는지
   - 트랜잭션이 유효한지
   - 앨리스 계좌에 1비트코인 이상 있는지
5. 검증 통과 시 블록체인에 추가

### 블록과 체인
- 200~300개의 트랜잭션이 하나의 **블록**을 구성
- **채굴자**가 트랜잭션을 모아 블록을 생성
- 새 블록을 다른 노드에 전파하여 검증받음
- 블록들이 이전 블록과 연결되어 **체인** 형성 = "블록체인"

### 프라이빗 키와 월렛
- **프라이빗 키**: 나만 알아야 하는 비밀 키 (금고 열쇠)
- 트랜잭션 발생 시 프라이빗 키로 서명
- **크립토 월렛**: 프라이빗 키를 안전하게 보관하는 도구
  - 시드 구문(Seed Phrase): 12개/24개 단어로 키를 생성/복구
  - 하드웨어 월렛: 10만원~수십만원 (프라이빗 키를 하드웨어에 안전 보관)
  - 소프트웨어 월렛: MetaMask 등 (편리하지만 상대적으로 덜 안전)

### 블록 익스플로러
- 모든 블록체인은 블록 익스플로러를 제공
- 주소를 입력하면 해당 주소의 모든 트랜잭션 이력을 확인 가능
- 상대방 주소를 검색하여 신뢰도를 판단할 수 있음

### 가스(Gas) 개념
- 트랜잭션을 보낼 때마다 수수료를 지불해야 함
- 서브스크립션이 아닌, 트랜잭션마다 비용 발생
- 과거(2017-2018): 트랜잭션당 수만 원~10만 원 이상
- 현재: 레이어 2 확장성 솔루션(예: 베이스체인) 등장으로 10원도 안 됨

## Part 3: 스마트 컨트랙트와 DApp

### 스마트 컨트랙트
- 블록체인 위에서 돌아가는 프로그램/코드
- 한번 올리면 영원히 블록체인 위에서 실행됨 (죽지 않음)
- 수정하려면 업그레이드 트릭 사용 (직접 수정은 불가)
- **탈중앙화된 프로그램**: 주인 없이 동작
- 주로 **Solidity** 언어로 작성

### 비탈릭 부테린과 이더리움
- 비탈릭 부테린: 블록체인에 범용 프로그래밍 기능을 추가한 이더리움 제안
- 비트코인은 디지털 화폐 목적
- 이더리움은 제너럴 퍼포스(범용 목적) 플랫폼

### DApp (탈중앙화 애플리케이션)
- 블록체인 위에서 동작하는 애플리케이션
- 예시: DeFi(탈중앙화 금융), DeSci(탈중앙화 과학), DeSoc(탈중앙화 소셜)
- 실제로 완전히 탈중앙화된 앱은 많지 않음

### 토큰
- **Fungible Token (대체 가능 토큰)**: 캐시와 같은 개념 (ERC-20)
  - 일정한 포맷에 맞춰 토큰 생성
  - 거래소 상장 시 가치 형성
- **NFT (Non-Fungible Token, 대체 불가 토큰)**: 소유 증명
  - 디지털 아트, 컬렉터블 외에도 다양한 소유 증명으로 활용 가능
  - 참석 증명, 인간 증명서 등으로도 활용 가능

- MetaMask 지갑 설치 및 사용
- 베이스체인(Base Chain) 테스트넷에서 실험

- 여러 종류의 블록체인을 MetaMask로 수용 가능

## 핵심 용어 정리

| 용어 | 정의 |
|------|------|
| **Immutable** | 불변성 - 한 번 기록되면 수정/삭제 불가 |
| **Transaction** | 거래 기록 (누가 누구에게 얼마를 보냈다) |
| **Node** | 블록체인 네트워크를 구성하는 개별 컴퓨터 |
| **Block** | 여러 트랜잭션을 묶은 데이터 단위 |
| **Mining (채굴)** | 트랜잭션을 검증하고 블록을 생성하는 과정 |
| **Private Key** | 개인 비밀 키 (서명 및 자산 접근에 사용) |
| **Wallet (월렛)** | 프라이빗 키를 보관하는 도구 |
| **Gas** | 트랜잭션 실행 수수료 |
| **Smart Contract** | 블록체인 위에서 동작하는 프로그램 |
| **DApp** | 탈중앙화 애플리케이션 |
| **Ledger** | 원장 - 중요한 트랜잭션을 기록하는 저장소 |
| **Censorship Resistance** | 검열 저항성 - 어떤 국가/기업도 통제 불가 |
| **Seed Phrase** | 시드 구문 - 12/24개 단어로 월렛 복구 |
| **Block Explorer** | 블록체인 트랜잭션을 조회하는 도구 |
| **Fungible Token** | 대체 가능 토큰 (ERC-20) |
| **NFT** | 대체 불가 토큰 - 고유한 소유 증명 |
| **Solidity** | 스마트 컨트랙트 프로그래밍 언어 |
| **Layer 2** | 블록체인 확장성 솔루션 (예: Base Chain) |
| **Permissionless** | 무허가 접근 가능 |
| **Composability** | 서비스 조합 가능성 |

## 강의 핵심 메시지

1. **블록체인은 직접 해보는 것이 가장 좋은 학습 방법**
2. 블록체인은 근본적으로 **신뢰 엔진(Trust Engine)** - 서로 못 믿는 당사자들 간의 신뢰를 제공
3. 블록체인의 5가지 핵심 특징: Immutable, Open, Shared, Permissionless, Distributed Ledger
4. 스마트 컨트랙트로 인해 블록체인 위에서 다양한 탈중앙화 서비스 구축 가능
5. NFT는 단순 디지털 아트가 아닌, 다양한 소유 증명으로 활용 가능
