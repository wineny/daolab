## 1. User Intent Discovery

### 1.1 Core Problem

CAVE 공통 설문 14문항이 5점 리커트 척도("전혀 아니다 ~ 매우 그렇다")를 사용하는데, 이 방식은:

1. **기분 의존적**: 같은 사람이 다른 날 응답하면 다른 점수가 나옴
2. **해석 모호**: "자율적으로 판단한다"의 3점이 사람마다 다른 의미
3. **중앙 편향**: 소표본에서 응답이 3점에 몰려 변화 감지 불가
4. **문항 중복**: Q1~Q3(의사결정), Q4~Q9(자율적 실행)에서 미세 변형이 과다

### 1.2 Target Users

| User Type | Usage Context | Key Need |
|-----------|---------------|----------|
| CAVE 조원 5명 | 각자 조직에서 before-after 측정 시행 | 일관된 데이터 수집, 쉬운 시행 |
| 조원의 팀원들 (n=5~20/조직) | 설문 응답자 | 짧고 명확한 문항, 부담 없는 응답 시간 |
| CAVE조 전체 | 5개 조직 교차비교 분석 | 표준화된 척도로 비교 가능한 데이터 |

### 1.3 Success Criteria

- [ ] 동일인이 1일 간격으로 응답해도 결과 편차가 기존 대비 감소 (검사-재검사 신뢰도 향상)
- [ ] 응답 시간 10분 → 7분 이하
- [ ] 5개 조직 교차비교 가능한 공통 구조 유지
- [ ] before-after 변화가 서술적으로 명확히 관찰 가능

### 1.4 Constraints

| Constraint | Details | Impact |
|------------|---------|--------|
| 조원 합의 필요 | 설문 변경은 5명 전원 동의 필요 (특히 에린 제안 문항 Q13-Q14) | High |
| Phase 1 시작 전 확정 | 4월 1주에 before 측정 시작 → 3월 내 확정 필요 | High |
| 기존 차원 보존 | 자율조직 4개 차원(의사결정분산/자율적실행/구성원인식/정보투명성) 매핑 유지 | Medium |
| 응답자 부담 | 팀원 대상 설문은 5~7분 이내여야 응답률 확보 가능 | Medium |

## 2. Alternatives Explored

### 2.1 Approach A: 빈도 기반 전환

| Aspect | Details |
|--------|---------|
| **Summary** | 모든 14문항을 "지난 1주간 몇 회?" 빈도 문항으로 전환 |
| **Pros** | 객관적, 기분 무관, before-after 변화 감지 용이 |
| **Cons** | 인식/태도 문항(Q10~Q12)은 빈도화 어려움 ("감시 느낌 횟수"는 부자연스러움) |
| **Effort** | Low |
| **Best For** | 행동 측정 가능한 문항이 대부분인 설문 |

### 2.2 Approach B: BARS(행동앵커) 전환

| Aspect | Details |
|--------|---------|
| **Summary** | 5점 리커트 유지하되, 각 점수에 구체적 행동 설명 부착 |
| **Pros** | 모든 문항에 적용 가능, 응답자간 해석 일치, 기존 척도 구조 유지 |
| **Cons** | 앵커 개발에 시간 소요, 조직별 맞춤화 필요할 수 있음 |
| **Effort** | Medium |
| **Best For** | 인식/태도 측정이 중요한 설문 |

### 2.3 Approach C: 하이브리드 (빈도 + BARS) — Selected

| Aspect | Details |
|--------|---------|
| **Summary** | 행동 측정 가능한 문항(Q1~Q5)은 빈도로, 인식/태도 문항(Q6~Q8)은 BARS로 |
| **Pros** | 문항 성격에 맞는 최적 방식 적용, 양쪽 장점 결합 |
| **Cons** | 척도 혼재로 분석 시 약간 복잡 |
| **Effort** | Medium |
| **Best For** | 행동+인식이 혼합된 조직 연구 설문 |

### 2.4 Decision Rationale

**Selected**: Approach C — 하이브리드 (빈도 + BARS)
**Reason**: CAVE 설문은 행동(의사결정, 자율적 실행)과 인식(감시 느낌, 투명성)이 혼합되어 있어, 단일 척도로는 최적화 불가. 빈도(행동)와 BARS(인식)를 문항 성격에 맞게 배치하면 모든 차원에서 신뢰도를 높일 수 있다.

## 8. Architecture Considerations

### 8.1 설문 구조 설계

이 프로젝트는 코드가 아닌 **설문 도구**이므로 기술 아키텍처 대신 설문 구조를 정의한다.

### 8.2 Key Decisions

| Decision | Options | Selected | Rationale |
|----------|---------|----------|-----------|
| 척도 방식 | 리커트 / 빈도 / BARS / 하이브리드 | 하이브리드 | 문항 성격별 최적 방식 적용 |
| 문항 수 | 14개(기존) / 10개 / 8개 / 6개 | 8개 | 4차원 커버 최소 문항 + 중복 제거 |
| 질적 보완 | 없음 / 이유한줄 / CIT / 다이어리 | 이유 한 줄 | 최소 부담으로 맥락 확보 |
| 빈도 범주 | 횟수 / 비율 / 혼합 | 횟수 (0/1-2/3-4/5+) | 직관적이고 회상 용이 |

### 8.3 개선 설문 전체 구조

```
CAVE 개선 설문 (8문항, ~7분)
│
├── 빈도 기반 (5문항) ─── "지난 1주간 몇 회?"
│   ├── Q1: 의사결정 분산 — 승인 없이 AI 활용 결정 횟수
│   ├── Q2: 자율적 실행(방식) — 도구/방식 직접 선택 횟수
│   ├── Q3: 자율적 실행(시간) — 순서/시간/일정 직접 조절 횟수
│   ├── Q4: AI 위임 허들 — 맡겨도 될 일 직접 한 횟수 + 이유
│   └── Q5: AI 위임 효과 — 맡겨서 안 해도 된 업무 건수
│
├── BARS 기반 (3문항) ─── 5단계 행동 앵커
│   ├── Q6: 자율성 인식 — 혼자 판단 범위 변화
│   ├── Q7: 감시 인식 — 들여다보여지는 느낌 정도
│   └── Q8: 정보 투명성 — AI 추천 이유 납득 정도
│
└── 질적 앵커 ─── 모든 문항 뒤 "이유 한 줄"
```

### 8.4 기존 → 개선 매핑

```
기존 14문항                          개선 8문항
─────────────────                  ─────────────────
Q1 (결정 자율)  ─┐
Q2 (방법 자율)  ─┼─ 통합 ──→       Q1 (승인없이 결정 횟수)
Q3 (승인 불필요) ─┘

Q4 (방식 선택)  ─┐
Q5 (도구 선택)  ─┼─ 통합 ──→       Q2 (도구/방식 선택 횟수)
Q6 (절차 개선)  ─┘

Q7 (순서 결정)  ─┐
Q8 (시간 배분)  ─┼─ 통합 ──→       Q3 (시간/일정 조절 횟수)
Q9 (일정 계획)  ─┘

Q13 (위임 허들) ──── 개선 ──→       Q4 (맡겨도 될 일 직접 횟수+이유)
Q14 (위임 효과) ──── 개선 ──→       Q5 (맡겨서 안한 업무 건수)

Q10 (자율성↑)  ──── BARS ──→       Q6 (판단 범위 변화, 5단계 앵커)
Q11 (감시 느낌) ──── BARS ──→       Q7 (들여다보여지는 느낌, 5단계 앵커)
Q12 (투명성)   ──── BARS ──→       Q8 (추천 이유 납득, 5단계 앵커)
```

| Intent Q3 | 개선 범위 | 기존 14문항 문구 개선 (구조 유지) | 차원 매핑 보존, 문구만 전환 |
| Alternatives | 빈도 vs BARS vs 하이브리드 | 하이브리드 (C) 선택 | 문항 성격별 최적 방식 |
| YAGNI | 14→8 통합, 주간펄스/비네트 제외 | 8문항 + 이유한줄 | 최소 부담, 최대 신뢰도 |

### BARS 기반 문항 (Q6~Q8)

**Q6. 자율성 인식 — 판단 범위 변화**
> AI 도입 이후, 혼자서 판단을 내릴 수 있는 범위가 어떻게 변했나요?

| 점수 | 행동 앵커 |
|------|----------|
| 1 | 전혀 변화 없음 — AI 도입 전과 판단 범위가 같다 |
| 2 | 약간 늘었음 — 단순한 업무 1~2개에서 혼자 판단하게 됐다 |
| 3 | 보통 — 일부 업무에서 AI 정보를 바탕으로 직접 결정한다 |
| 4 | 꽤 늘었음 — 이전에 상사에게 물어봤던 결정을 이제 스스로 한다 |
| 5 | 크게 늘었음 — AI 덕분에 팀에 새로운 방식을 제안할 수준이 됐다 |

> → 위 점수를 준 이유 (구체적 사례): _______________

**Q7. 감시 인식 — 들여다보여지는 느낌** (역방향)
> AI가 업무를 추적/관리하면서, 들여다보여지는 느낌은 어느 정도인가요?

| 점수 | 행동 앵커 |
|------|----------|
| 1 | 전혀 없음 — AI는 도구일 뿐, 감시와 무관하다 |
| 2 | 약간 — AI가 기록을 남기지만 신경 쓰이지 않는다 |
| 3 | 보통 — AI 추적 데이터를 누가 볼지 가끔 의식된다 |
| 4 | 꽤 느낌 — AI 기록이 평가에 쓰일 수 있다는 생각이 든다 |
| 5 | 강하게 느낌 — AI가 나를 모니터링하는 느낌이 있다 |

> → 위 점수를 준 이유 (구체적 사례): _______________

**Q8. 정보 투명성 — AI 추천 이유 납득**
> AI가 추천하거나 정리한 것의 이유를 납득할 수 있나요?

| 점수 | 행동 앵커 |
|------|----------|
| 1 | 전혀 — AI 결과물을 왜 그렇게 냈는지 모르겠다 |
| 2 | 약간 — 가끔 맞는 것 같지만 근거는 불분명하다 |
| 3 | 보통 — 대체로 이해하지만 확인이 필요할 때가 있다 |
| 4 | 꽤 — AI 추천의 근거를 대부분 이해하고 신뢰한다 |
| 5 | 완전히 — AI의 판단 과정을 파악하고, 틀린 부분도 짚어낼 수 있다 |

> → 위 점수를 준 이유 (구체적 사례): _______________

## Appendix C: 리서치 근거

### 적용한 방법론

| 방법론 | 출처 | 적용 |
|--------|------|------|
| BARS (Behavioral Anchored Rating Scale) | Smith & Kendall (1963) | Q6~Q8 행동 앵커 |
| 빈도 기반 척도 | Behavioral observation 전통 | Q1~Q5 행동 빈도 측정 |
| 질적 앵커링 (Mixed Methods) | Creswell & Plano Clark (2018) | 모든 문항 "이유 한 줄" |
| 역코딩 (Reverse coding) | Podsakoff et al. (2003) | Q4(허들), Q7(감시) |
| 1주 회상 기간 | ESM/EMA 연구 전통 | 빈도 문항 recall window |

### 검토 후 제외한 방법론

| 방법론 | 제외 이유 |
|--------|----------|
| ESM/EMA (일일 다회 측정) | 2개월 연구에서 응답 피로, 기술 인프라 부재 |
| 강제선택 (Forced-choice) | 절대 수준 측정 불가로 before-after 부적합 |
| WDQ 전면 도입 | 기존 CAVE 프레임 변경 범위 초과 |
| 비네트 (Vignette) | 효과적이나 설문 시간 증가, 향후 보충 도구로 검토 |
| 6점 척도 (중앙점 제거) | BARS에서는 5점이 자연스러움, 빈도는 범주형이라 무관 |
