| 관점 | 내용 |

## 1. User Intent Discovery

### 핵심 문제
- 봇이 1회성 Q&A로 끝남 → 대화 맥락 유지 안 됨 (매번 @멘션 필요)
- 32명 멤버가 서로의 관심사/배경을 파악하기 어려움
- 지식파일 업데이트가 수동 → 시간 지나면 낡은 정보

### 대상 유저
- 7기 멤버 32명 (1차 대상)

### 성공 기준
- 스레드에서 멘션 없이 자연스럽게 봇과 대화 가능
- "나랑 비슷한 관심사 가진 사람?" 질문에 정확한 매칭 제공
- 멤버가 공유한 자료가 봇 지식에 반영되어 답변에 활용

## 2. Alternatives Explored

| 접근법 | 설명 | 선택 |
|--------|------|------|
| **A: OpenClaw 네이티브 활용** | skill/memory/heartbeat + 디스코드 네이티브 기능 조합 | ✅ 선택 |
| B: 외부 연동 (노션/시트) | MCP로 Notion DB 연동, 구조화된 데이터 | 향후 검토 |
| C: 게이미피케이션 | 포인트/배지/리더보드 | 보류 (외적보상→내적동기 감소 리스크) |

**선택 이유**: 추가 인프라 없이 OpenClaw 기능만으로 즉시 적용 가능. 다오랩의 "완벽보다 시도" 원칙에 부합.

## 3. Architecture

### 3-1. 전체 구조

```
Discord
  ├─ 서버 채널: @멘션으로 대화 시작
  ├─ 스레드: 멘션 없이 이어서 대화
  ├─ DM: 자유 대화
  └─ 멘션+파일/링크: 지식 수집
       │
       ▼
OpenClaw (daolab agent)
  ├─ SOUL.md (페르소나+보안)
  ├─ knowledge/ (6개 지식파일)
  ├─ memory/ (수집된 지식 저장)
  └─ skills/ (커스텀 명령어)
       ├─ 멤버 검색/매칭
       └─ 지식 수집/관리
```

### 3-2. 기능별 구현 방식

#### 기능 1: 스레드 대화 지원
- **구현**: OpenClaw Discord 설정에서 thread 지원 활성화
- **동작**: @Daolab 멘션 → 봇 답변 → 스레드에서는 멘션 없이 대화 계속
- **설정**: `chatTypes: thread` 활용, 스레드 내 메시지는 자동 응답

#### 기능 2: 멤버 프로필 조회/매칭
- **구현**: 03_members_summary.md 데이터를 활용한 자연어 검색
- **동작 예시**:
  - "개발자 누가 있어?" → 개발 관련 멤버 목록
  - "나랑 비슷한 관심사 가진 사람?" → (질문자 프로필 기반) 매칭
  - "커뮤니티 운영 경험자?" → 해당 멤버 추천
- **데이터**: 이미 03_members_summary.md에 직업/관심주제/참여농도 정보 있음
- **강화**: 멤버별 관심 키워드 태그 추가하여 매칭 정확도 향상

#### 기능 3: 지식 자동 업데이트
- **구현**: heartbeat 또는 cron으로 정기적 지식파일 갱신 체크
- **동작**: 일정 변경, 새로운 공지 등을 반영
- **방식**: 운영자가 knowledge/ 파일을 업데이트하면 다음 세션에 자동 반영 (현재도 작동)
- **향후**: 노션/구글닥스 MCP 연동으로 자동 동기화

#### 기능 4: 사용자 제공 지식 수집
- **구현**: 멘션 + 파일/링크 감지 → memory/에 저장
- **동작**:
  - `@Daolab 이거 기억해줘 [파일/링크/텍스트]` → 지식으로 저장
  - `@Daolab 뭐 기억하고 있어?` → 수집된 지식 목록
- **저장**: memory/ 폴더에 날짜별 또는 주제별로 저장
- **보안**: 저장된 내용은 다오랩 관련 질문 답변에만 활용

## 5. Implementation Steps

### Step 1: 스레드 대화 설정
- OpenClaw Discord 설정에서 thread 응답 활성화
- 스레드 내 메시지에 자동 응답하도록 구성
- 테스트: 서버 채널 → 멘션 → 스레드 → 멘션 없이 대화

### Step 2: 멤버 매칭 강화
- 03_members_summary.md에 관심 키워드 태그 추가
- SOUL.md에 멤버 매칭 답변 규칙 추가
- 테스트: "AI 관심 있는 사람?", "커뮤니티 운영 경험자?" 등

### Step 3: 지식 수집 기능
- SOUL.md에 지식 수집 규칙 추가
- memory/ 폴더 활용한 지식 저장 구조 설계
- "@Daolab 이거 기억해줘" 패턴 인식
- 테스트: 파일/링크/텍스트 수집 및 활용

### Step 4: 통합 테스트
- 전체 시나리오 테스트
- 보안 테스트 (정보 유출 방지)
- 7기 멤버 대상 안내

## 7. Risks & Mitigations

| 리스크 | 대응 |
|--------|------|
| 스레드 자동 응답이 OpenClaw에서 지원 안 될 수 있음 | Discord API 직접 설정 또는 reply-chain 방식으로 대체 |
| 수집된 지식이 토큰 제한 초과 | memory/ 파일 크기 모니터링, 주기적 요약/정리 |
| 잘못된 정보가 지식으로 수집될 수 있음 | 수집 시 확인 메시지, 관리자만 삭제 가능 |
| 멤버 매칭 정확도 | 키워드 태그 + 자연어 이해로 보완 |
